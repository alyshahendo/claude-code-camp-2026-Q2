# 01 · The Struct Skeleton (Python)

Python port of the Ruby step 1 struct skeleton.

Let's define data structures for specific data we need to constantly pass around.
- `Boukensha.Tool`
- `Boukensha.Message`
- `Boukensha.Context`

Ruby represents these as `Struct`s because they are lightweight and make the code
readable for learning. Python has no direct `Struct` equivalent, so each is a
small plain class with `__init__` and a `__str__` that mirrors the Ruby output.
In practice we would probably reach for `@dataclass`, but plain classes keep this
port consistent with the step 0 style.

## Data Structures

### `boukensha.Tool`

| Field | Description |
|---|---|
| `name` | The name of the tool |
| `description` | Shown to the agent so it knows when to invoke the tool |
| `parameters` | The arguments that need to be passed in |
| `block` | The callable that runs when the tool is called |

```
#<Tool name=move description=Move the player in a direction (nor params=['direction']>
```

### `boukensha.Message`

A single unit of conversation between the user and the agent.

| Field | Description |
|---|---|
| `role` | Who is speaking — user, assistant, or tool_result |
| `content` | The text generated by the agent or provided by the user |
| `tool_use_id` | Links a tool result back to the specific tool call that requested it |

```
#<Message role=user content=Explore north and tell me what you find....>
#<Message role=assistant content=Sure, let me head north and take a look....>
```

### `boukensha.Context`

Holds everything Boukensha needs to make an API call. Nothing lives outside of this.

| Field | Description |
|---|---|
| `system` | Tells the agent what its job is; sent as a separate API field, never inside the messages array |
| `messages` | The full conversation history, replayed on every turn |
| `tools` | The registered tools the agent is allowed to invoke |

```
#<Context task=player turns=2 tools=1>
```

## Layout

| File | Purpose |
|------|---------|
| `boukensha/config.py` | `Config` class (from step 0) |
| `boukensha/tool.py` | `Tool` data structure |
| `boukensha/message.py` | `Message` data structure |
| `boukensha/context.py` | `Context` data structure |
| `boukensha/tasks/base.py` | abstract `Base` (provider/model + prompt resolution) |
| `boukensha/tasks/player.py` | concrete `Player` (the main loop) |
| `boukensha/__init__.py` | top level exports |
| `prompts/system.md` | default system prompt shipped with the library |
| `examples/example.py` | runnable smoke test |

## Run Example

./week1_baseline/bin/python/01_struct_skeleton
